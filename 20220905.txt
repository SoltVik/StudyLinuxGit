(1) Что делает команда git revert и пример ее использования.
	Отменяет изменение, путём создания нового коммита. Создаёт изменения обратные отменяемому коммиту. Пример импользования: сделали правки и запушили их. Как выяснилось позже, этот коммит нарушает работу некоторых сервисов. Чтобы вручную не прописывать необходимы изменения для отмены коммита, то можно использовать команду git revert.
(2) Какой командой можно изменить последний коммит?
	git commit --amend
(3) Что делает команда git remote rm origin?
	Убирает привязку к удалённому репозиторию
(4) В каких случаях происходит MERGE CONFLICT?
	Если при слиянии откличается код файла в одной и той же строке. Пример: в ветки А изменили файл 1 в 15 строке. Так же в ветки Б изменили файл 1 в 15 строке и изменения отличаются от изменений в ветке А, то при слиянии возникнит конфликт.
(5) Как посмотреть отличия между ветками?
	git diff BRANCH1 BRANCH2
(6) Какой командой удалить локальную и удаленную ветки?
	git branch -d BRANCH_NAME --> далить локальную ветку
	git push origin --delete BRANCH_NAME --> удалить ветку из удалённого репозитория
(7) Как сделать файл неотслеживаемым (untracked)?
	git rm --cached FILE_NAME --> снимаем файл с учёта, при этом сам файл не удаляется.
